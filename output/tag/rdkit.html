Exploring DrugBank using RDKit

Published: Wed 08 June 2016
By Daniel P. Russo

In Python.

tags: rdkit python drugbank

For a blog titled Drug Discovery in Python, it's probably fitting that I should have at least one blog post dealing with drugs and/or Python. So with that, let me introduce a package I use now on a daily basis called RDKit. RDKit is open-source cheminformatics software available with a Python API. I first discovered RDKit sometime in the Fall of 2015 and quickly found it to be very powerful. Unfortunately, I had problems installing it on my Windows computer which is where I do most of my programming. Installation on Linux (for whatever reason) was flawless. Fortunately, the latest release version 2016.03.1 installed fine on my Windows machine using Anaconda. If you are using the Anaconda distribution of Python you can install the latest version of RDKit with the following command:

conda install -c rdkit rdkit=2016.03.1

For those interested, Greg Landrum's blog showcases RDKit experiments and tutorials. For this post, I will briefly introduce RDKit functionality using chemicals from a database called DrugBank which contains 8,206 drug-like compounds, hosted by the Wishart Research Group in the University of Alberta. These compounds are available for download from their website in structure-data file (SDF) format (structures.sdf).

First things first, we import RDKit.

In [1]:
from rdkit import Chem

Now load in the DrugBank molecules. We can read compounds directly from the SDF file, creating a sequence of Mol objects.

I have no idea what the above compound is. But, luckily, all the data associated with compounds in the SDF file are retained as properties of the RDKit Mol. We can find a particular property by calling the GetProp() function.

In [22]:
drugbank[0].GetProp('GENERIC_NAME')
Out[22]:
'Bivalirudin'

Well, still not sure what 'Bivalirudin' is, but according to wikipedia:

Bivalirudin (Angiomax or Angiox, manufactured by The Medicines Company) is a specific and reversible direct thrombin inhibitor (DTI).

Chemically, it is a synthetic congener of the naturally occurring drug hirudin (found in the saliva of the medicinal leech Hirudo medicinalis).

So, it's an anticoagulant from leeches. If the term ethnobotany refers to the study of compounds from medicinal plants, what's the term to study compounds from medicinal parasites?

Anyway, leeches aside, there are plenty more properties contained within a DrugBank SDF file (now an RDKit Mol). We can access them using the GetPropNames() function, which returns a tuple of properties.

Looks like a lot of the classics are there. Both computed and experimental properties. However, usually I am most interested in chemically similar compounds. We use can use Bivalirudin as a case study to find the approved drugs in DrugBank that are most chemically similar.

In [8]:
approved = []
for drug in drugbank:
    if drug and 'approved' in drug.GetProp('DRUG_GROUPS'):
        approved.append(drug)
len(approved)
Out[8]:
1852

Of the 1,852 compounds, we can create the standard MACCS stuctural fingerprints and calculate the similarity using the tanimoto coefficient. After creating the MACCS fingerprints, we can get the tanimoto coefficient in bulk.

In [9]:
from rdkit.Chem import AllChem
from rdkit import DataStructs

bivalirudin_fp = AllChem.GetMACCSKeysFingerprint(drugbank[0])
fps = [AllChem.GetMACCSKeysFingerprint(mol) for mol in approved]
sims = DataStructs.BulkTanimotoSimilarity(bivalirudin_fp, fps)
sims[:10]

And of course data analysis is so much easier in Pandas, so let's use that to find our analogues and see the compounds that have a similarity of 0.8 or greater.

In [11]:
import pandas as pd

similarities = pd.Series(sims, index=approved)
similarities.sort_values(ascending=False, inplace=True)

highest_matches = similarities[similarities > 0.8]
len(highest_matches)
Out[11]:
9

Now, we can plot these structures in a nice grid format with the name and similarity of each compound to Bivalirudin.

In [12]:
from rdkit.Chem import Draw
legends = []
for mol, sim in highest_matches.iteritems():
    # concat string for neat printout
    s =  "{0} : {1:.2}".format(mol.GetProp('GENERIC_NAME'), sim)
    legends.append(s)
Draw.MolsToGridImage(highest_matches.index.tolist(), 
                     molsPerRow=4, legends=legends)
